# Contributing to Workspace Chat Platform

First off, thank you for considering contributing to the Workspace Chat Platform! It's people like you that make it such a great tool.

---

## 🏗️ Development Workflow

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Branch**: Use descriptive names like `feature/new-button` or `bugfix/login-error`.
3.  **Implement Changes**: Follow our [Development Guide](./development.md) for coding standards.
4.  **Test Your Changes**: Ensure no regressions are introduced.
5.  **Commit**: Use [Conventional Commits](https://www.conventionalcommits.org/).
6.  **Push & Pull Request**: Submit your changes for review.

---

## 📜 Coding Standards

- **TypeScript**: Strict typing is mandatory.
- **Linting**: Run `npm run lint` before committing.
- **Formatting**: Run `npm run format` to ensure consistent style.
- **Components**: Follow the existing Geist-inspired design tokens in [docs/design.md](./design.md).

---

## 🐞 Reporting Bugs

- Use the GitHub Issue tracker.
- Describe the bug clearly.
- Provide steps to reproduce the issue.
- Include environment details (Node version, OS, Browser).

## 💡 Feature Requests

- Clearly explain the problem the feature solves.
- Describe your proposed solution.
- Be open to feedback and discussion.
